Introduction au SQL

***********************************

QUELQUES DEFINITIONS:

Une base de données : Il s'agit  d'un emplacement de données sauvegardées. La BDD inclut des tables qui sont composées de champs ou colonnes et qui contiennent des enregistrements correspondant aux lignes.

SGBD : Système de gestion de bases de données : c'est un logiciel qui sert d'interface entre utilisateur et BDD permettant : d'introduire des données, de les mettre à jour et d'y accéder.

SQL : Structured Query Langage : Il s'agit d'un système de requêtes qui permet d'interroger une base de données.


*********************************

SGBDR et SGBD NoSQL :

On distigue deux principaux SGBD, à savoir :
  - les SGBDR pour système de gestion de bases de données relationnelles : par exemple MySQL
    il consiste à stocker l'information dans des tables définies par des colonnes, des clés primaires, des clés étrangères. cela permet de ne pas stocker l'information plusieurs fois, mais que dans une seule table.

 - Les NoSQL : Not only SQL :  Cela ressemble fortement à un grand tableau unidimensionnel avec des millions de données. Il n'y a plus de colonnes, on parle de bases de données clé-valeur.

 => exemples de SGBDR :
    MySQL
    Oracle
    Microsoft SQL Server
    PostgreSQL

=> exemples de SGBD NoSQL :
    Mongo DB
    Redis
    CouchDB
    Cassandra

*********************************

Deux exemples d'utilisation des BDD et du SQL dans des sites dynamiques :
    => Inscription / Connexion d'un membre du site
    => Site e-commerce avec une page boutique et une page détaillée du produit.

Un site dynamique, par opposition à un site statique, est caractérisé par la présence d'un Front-office, partie visible par l'internaute, d'un  Back-office, partie dédiée à l'administration du site, et d'une base de données.
